# Як зібрати OLX API асистента в Claude.ai (5 хвилин)

## Що ти отримаєш

Project у Claude.ai під назвою «OLX API Assistant», в який можна заходити з будь-якого пристрою (десктоп, мобільний, web) і запитувати все про OLX Partner API. Асистент відповідає українською, знає всі ендпоінти, помилки і правила платформи, і враховує специфіку нерухомості та твій DFY-бізнес.

---

## Крок 1. Створити Project

1. Зайди в [claude.ai](https://claude.ai).
2. У лівому сайдбарі знайди розділ **Projects** → натисни **«+ Create project»** (або «New project»).
3. Назва: `OLX API Assistant`.
4. Опис (необов'язково): `Спеціалізований асистент з інтеграції OLX Partner API v2.0 для нерухомості UA`.
5. Натисни **Create**.

## Крок 2. Завантажити Knowledge Base

1. Усередині нового Project знайди секцію **Project knowledge** (або «Files», залежно від версії UI).
2. Натисни **«+ Add content»** → **«Upload files»** (або просто перетягни файл).
3. Завантаж файл `02_KNOWLEDGE_BASE.md`.
4. Дочекайся, поки Claude проіндексує — зазвичай 5–10 секунд.

## Крок 3. Вставити інструкції проекту

1. У Project знайди кнопку **«Set custom instructions»** (іконка налаштувань або кнопка зверху, залежно від UI).
2. Відкрий файл `01_PROJECT_INSTRUCTIONS.md`.
3. Скопіюй **весь текст від першого рядка `---` і нижче** (тобто без заголовка «Project Instructions — OLX Partner API Assistant» і службового рядка «Куди це вставити»).
4. Встав у поле інструкцій.
5. Збережи.

## Крок 4. Перевірити

Створи новий чат всередині Project і запитай щось типового:

> Покажи мені приклад POST `/adverts` для продажу 2-кімнатної квартири в Дніпрі.

Якщо асистент:
- ✅ відповідає українською;
- ✅ дає готовий JSON з `category_id`, `district_id`, `attributes`;
- ✅ посилається на «KB §6» або інший розділ knowledge base;
- ✅ додає список типових помилок з §7;

— все працює правильно.

Якщо відповідь загальна, без посилань на KB — щось пішло не так з завантаженням файлу. Видали і перезавантаж `02_KNOWLEDGE_BASE.md`.

## Що ще можна зробити (опційно)

### Додати референс-дані по Дніпру

Зараз у KB категорії та district_id вказані як приклади. Щоб асистент давав готові цифри, потрібно один раз витягти:

```http
GET https://www.olx.ua/api/partner/categories
GET https://www.olx.ua/api/partner/cities/{dnipro_city_id}/districts
GET https://www.olx.ua/api/partner/categories/{rent_apartment_id}/attributes
GET https://www.olx.ua/api/partner/categories/{sale_apartment_id}/attributes
```

Зберегти відповіді в окремий файл `03_REFERENCE_DATA_DNIPRO.md` і теж завантажити в Project knowledge. Тоді асистент буде підставляти реальні ID у шаблони запитів.

Це робиться один раз, після того як отримаєш `client_id`/`client_secret` від OLX.

### Налаштувати на мобільному

- Зайди в додаток Claude на iOS/Android під тим самим акаунтом.
- Project автоматично з'явиться в списку.
- Можна питати голосом (через диктовку) — це добре стикується з твоїм робочим стеком (Obsidian + AquaVoice).

### Розширення під DFY-бізнес

Коли почнеш реалізовувати MVP, додавай у Project knowledge:
- Архітектурні рішення (схеми, ADR);
- Зразки коду інтеграції;
- Помилки і їх причини, які знаходив на практиці;
- Виписки з листування з OLX support.

З часом цей Project стане твоєю інженерною пам'яттю по OLX-інтеграції.

## Якщо щось зламається

- **Асистент відповідає англійською** → перевір, чи інструкції з `01_PROJECT_INSTRUCTIONS.md` повністю скопіювалися в поле Custom instructions.
- **Асистент не посилається на KB** → перевір, що файл `02_KNOWLEDGE_BASE.md` дійсно завантажений у Project knowledge (іноді треба оновити сторінку).
- **Асистент придумує ендпоінти** → перевір, що в інструкціях є фраза «не вигадувати ендпоінти, яких немає в KB». Це є в шаблоні, але якщо ти редагував — міг видалити.
- **Хочу оновити KB** → просто видали старий файл з Project knowledge і завантаж нову версію.
